Issues

Issue 1: "app.use(function(req, res, next) { res.header("Access-Control-Allow-Origin", "*"); res.header("Access-Control-Allow-Headers", "Origin, X-Requested-With, Content-Type, Accept"); next(); });"

When cross-origin resource sharing is always in effect as is show by the "*" inclusion on line 10 in his index.js file, having cors enacted all the time can raise vulnerabilities. This is is true because the requested resource allows sharing at every origin, and that means any site can send an XHR request and access his server's response which is a security risk.

This exploits vulnerabilities in the code andallows unwanted script to be inserted when a curl is executed. Hackers can query the database and inject script which can be a high risk vulnerability.
